Understanding the Segmentation

Standard averages fail when data is extremely skewed. With a skewness score of 25.33, using simple averages would suggest a typical user spend that doesn't reflect reality for the vast majority of customers, nor does it capture the immense value of the top performers.

  • Platinum (Mega Whales, 1.5%): The highest revenue drivers requiring white-glove VIP support
  • Gold & Silver (Middle Tiers): Targets for upsell campaigns to move them into higher value segments
  • Starter & Copper (Base Tiers): Best served by automated onboarding and self-service tools

Methodology

The Challenge: Extreme Skewness

Initial diagnosis revealed an extreme skewness score of 25.33. On a linear scale, 99% of users appeared compressed against the axis, hiding the "Whale Zone" where the bulk of revenue actually sits.

Segmentation Approach

  • Data Cleaning: Processed 85,099 raw user records, removing invalid entries (zero/negative spend)
  • Distribution Analysis: Used log-scale visualization to uncover the hidden structure of user spend
  • Revenue Density Analysis: Weighted histogram showing where the money actually is
  • Tier Identification: Grouped users into 6 distinct tiers based on Net Merchandise Value (NMV)
  • Validation: Confirmed the "Pareto Principle on steroids" - 1.5% driving 81% of value

Core Algorithm (Python) 
# Log-scale transformation for skewed data
import numpy as np
import pandas as pd

# Calculate skewness
skewness = df['DAY_AVG_NMV'].skew()
print(f"Skewness: {skewness:.2f}")  # 25.33

# Apply log transformation
df['log_nmv'] = np.log10(df['DAY_AVG_NMV'] + 1)

# Define tier boundaries based on log-scale analysis
tier_boundaries = {
    'Starter': (0, 10),
    'Copper': (10, 25),
    'Bronze': (25, 50),
    'Silver': (50, 100),
    'Gold': (100, 250),
    'Platinum': (250, np.inf)
}

# Assign tiers to users
def assign_tier(nmv):
    for tier, (low, high) in tier_boundaries.items():
        if low <= nmv < high:
            return tier

df['tier'] = df['DAY_AVG_NMV'].apply(assign_tier)

Scope & Limitations

What This Is NOT

  • Not Predictive: This is descriptive segmentation based on historical data, not a predictive model
  • Static Snapshot: Tier boundaries should be recalculated as the business grows and customer behavior evolves
  • Single Feature: Uses only NMV; could be enhanced with purchase frequency, product categories, customer tenure, etc.

Future Enhancements

  • Multi-dimensional Clustering: K-means or DBSCAN on multiple features (frequency, recency, monetary)
  • Time-series Analysis: Track user migration between tiers over time to identify growth patterns
  • Predictive Modeling: Build models to predict which users are likely to move up or down tiers
  • A/B Testing: Validate business impact of tier-specific interventions and measure ROI
