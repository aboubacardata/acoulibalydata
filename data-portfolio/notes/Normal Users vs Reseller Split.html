Project: Statistical Segmentation of Resellers vs. Normal Users¶

1. Problem Statement¶

Our platform has a diverse user base ranging from casual individual buyers to high-volume commercial resellers. However, we currently lack a rigorous, data-driven method to distinguish between these two groups.

Simple "average" metrics fail because the data is highly skewed a small number of power users distort the mean. Without a clear separation, we cannot:

  • Tailor support (e.g., VIP lanes for resellers).
  • Detect policy abuse.
  • Accurately forecast revenue risks.

The Goal: To identify a statistically robust threshold for DAY_AVG_NMV (Daily Average Net Merchandise Value) that separates "Normal Users" from "Resellers" using unsupervised anomaly detection techniques.

2. Methodology & Key Assumptions¶

To solve this, we use the Box-Cox Transformation, a statistical technique designed to normalize highly skewed data.

Key Assumptions:

  1. NMV as a Proxy: We assume DAY_AVG_NMV is the strongest signal for reseller intent. While purchase frequency matters, daily volume is the primary differentiator for commercial activity.
  2. Lognormal-like Distribution: User spending data typically follows a "power law" or lognormal distribution (many small spenders, few whales). We assume that by transforming this data into a normal (bell curve) shape, we can use Standard Deviations (SD) to identify outliers.
  3. The "1.5 Sigma" Rule: We define a "Reseller" as anyone falling more than 1.5 Standard Deviations above the mean in the transformed space. This is a conservative threshold designed to isolate the statistically significant "tail" of the distribution.
  4. Positive Data Constraint: Box-Cox requires strictly positive data. We assume that adding a negligible shift ($+0.0001$) to handle zeros does not distort the behavior of high-value users.

3. Scope & Limitations (The "What Not")¶

It is important to define what this analysis is not doing:

  • Not a Classification Model: This is an unsupervised segmentation. We are not training a model on "known" resellers (labeled data); we are discovering them based on statistical divergence.
  • Static Snapshot: The threshold is calculated based on the current dataset's distribution. If the business grows significantly, this threshold should be recalculated.

# 1. Apply Box-Cox Transformation
data_transformed, best_lambda = stats.boxcox(df['NMV_Shifted'])
mean_trans = np.mean(data_transformed)
std_trans = np.std(data_transformed)

# 2. Helper Function: Inverse Box-Cox
def inv_boxcox(y, ld, shift):
    if ld == 0: return np.exp(y) - shift
    else: return (y * ld + 1)**(1/ld) - shift

# 3. Calculate All Potential Thresholds for the Table
sd_multipliers = [1, 1.5, 2, 3]
comparison_data = []

for mult in sd_multipliers:
    limit_trans = mean_trans + (mult * std_trans)
    limit_orig = inv_boxcox(limit_trans, best_lambda, shift_value)
    
    comparison_data.append({
        'SD Cut-off': f'{mult} SD',
        'Threshold ($ NMV)': limit_orig,
        'Resellers (#)': len(df[df['DAY_AVG_NMV'] > limit_orig]),
        'Resellers (%)': (len(df[df['DAY_AVG_NMV'] > limit_orig]) / len(df)) * 100,
        'Revenue Captured (%)': (df[df['DAY_AVG_NMV'] > limit_orig]['TOTAL_NMV'].sum() / df['TOTAL_NMV'].sum()) * 100
    })

logic_df = pd.DataFrame(comparison_data)

# Apply the chosen segment
df['Segment'] = np.where(df['DAY_AVG_NMV'] > chosen_threshold, 'Reseller', 'Normal')

# Aggregate
summary = df.groupby('Segment').agg(
    User_Count=('DAY_AVG_NMV', 'count'),
    Total_Revenue=('TOTAL_NMV', 'sum')
).reset_index()
